  • Figure 1 shows the arena design for the project.

  • The arena is an abstraction of a parking area made up of a parking slots and path for the vehicles.

  • There are total 40 Parking slots available in the arena (Marked by red colored numbers).


    Figure 1: Arena design

1. Arena Components

  • The arena consists of following components:
    • Entry Gates: There are 4 entry gates marked by red colored blocks. They consist of sensors to count the number of vehicles passing through them.
    • Self Park Vehicles: These Self Park vehicles are the bots which were used in Lab 4 and Lab 5. There can be minimum 3 Self Park vehicles in the arena, maximum can be 6.
    • Obstacles: The Obstacles are dummy vehicles which are present in the path of the arena marked by orange color.
    • Full Slots: Full Slots are the parking slots which are occupied by the dummy vehicle as shown in the arena.
    • Reserved Slots: Now the Reserved slots as the name suggests are the parking slots which cannot be used by any vehicle.
    • Empty Slots: The Parking slots which are not full or reserved are considered to be Empty Slots.
    • Parking Slot Numbers: The Parking Slot Numbers shows the number of the parking slots those are there in each lane of the arena.

Figure 2: Terms associated with Arena

2. Given Inputs:

  • A text file namely vehicle_handler.txt will be provided to the teams which will contain following parameters:
    • Starting time of the Self Park Vehicle (Ti)
    • Time duration within which Vehicle should Park (Ci)
    • Leaving time of the Self Park Vehicle (Di)
    • Entry gate number from which Self Park Vehicle should pass (Ei)

3. Web App Interface:

  • The Web App Interface should depict various actions which are happening in the Coppeliasim.
  • It will act as server for providing the data from vehicle_handler.txt file to the Coppeliasim.
  • It should show following things:
    • A timer from the start of the run.
    • The Empty, Reserved and Full parking slots obtained from Coppeliasim at the start of the run using available sensors (Teams are free to add any number of sensors).
    • Traversal of Self Park Vehicle from node to node.
    • Entry and Exit of the Self Park Vehicle.

4. Theme Run Sequence:

  • By using any of the sensors available in Coppeliasim find out the Empty, Reserved and Full parking slots in given configuration and communicate it to the server.
  • The server will also read the data from given vehicle_handler.txt file and communicate appropriate information to the Coppeliasim.
  • The ith Self Park Vehicle will start at Tith time from the entry gate Ei present in the arena.
  • Self Park Vehicle must traverse around the arena avoiding Obstacles and park itself in any of the empty slots within its respective Ci duration.
  • The Self Park Vehicle must park itself for Di time duration and after that time it must leave the parking area from any of the entry gates.
  • The entire run should be reflected on the server side and should provide the visual demonstration of the actions using webapp interface.

5. Communication Sequence

  • Coppeliasim to Server:
    • Get the locations of Empty, Reserved and Full parking slots as per given configuration and send it to server for showing in the Web App Interface.
    • Location of Self Park Vehicles from node to node.
  • Server to Coppeliasim:
    • Read the vehicle_handler.txt file and send the data from server to the Coppeliasim.
