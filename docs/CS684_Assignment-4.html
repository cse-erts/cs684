CS684: Embedded System Course

Assignment 4: FreeRTOS/scheduling theory

AIM

The objective of this assignment is to incorporate RTOS features into the current project. By utilising capabilities such as Scheduling, Interrupt Handling, Memory management, and Task management, the project can achieve improved response time and execution.

Description

  • You can determine which tasks in the existing project you are working on could benefit from being run as independent tasks with their own threads.
  • Once decided students need to implement the task logic and integrate it with the existing supervisor.
  • You can install the FreeRTOS library from Arduino. Open the Arduino IDE, and click sketch -> Include Library -> Manage Libraries, and choose the FreeRTOS file.
  • The examples given for FreeRTOS will give a clear understanding of how the codes can be written.
  • Following are some of the examples of how you can use the FreeRTOS features:
    • Think of one or more individual tasks that are there in the entire project (Search & Rescue or Valet Parking). Tasks can be line following, colour sensing, object following, motion control using PWM, ADC conversion, etc.
    • Prioritise tasks based on the importance and criticality it has to the project. Using FreeRTOS’s Task management feature you can make sure highest priority is given to specific tasks which are to be executed first.
    • Inter-task Communication can be used to communicate with different tasks of projects while sharing resources.
    • Interrupt handling can be used to detect and respond to external events in the real time.

Submission Instructions

  1. For this assignment submission you have to upload a .tar.gz file.
  2. Folder should contain following: (You can implement the logic using either Statechart or Heptagon)

  • For Statechart: Statechart Project folder: It should contain following

    • Logic.ysc file: containing the statechart implementation of the logic.
    • Logic.sgen file: for generating the c code from statechart.
    • Supervisor Folder: containing
      • supervisor.ino file written using FreeRTOS
      • and auto generated c files from the statechart

  • For Heptagon: Heptagon Project folder: It should contain following

    • Logic.ept file: containing the heptagon implementation of the logic.
    • Integrate.sh file: to generate the c files from heptagon.
    • Supervisor Folder:
      • supervisor.ino file written using FreeRTOS
      • and auto generated c files from the heptagon

  • Readme.txt file:

    • The problem statement outline the tasks done using FreeRTOS
    • Listing the points on how the tasks are optimised because of FreeRTOS features
    • Also add the youtube link of the video (implementation of problem statement using hardware)
  1. Compress the folder and rename it as <RollNo>_assignment4.tar.gz.
  2. Upload the file on Moodle.
